1.4 릴레이션 특징.
속성은 
   단일값.
   서로 다른 이름.    
   한 속성 같은 다 같은 도메인 값.
   순서는 상관이 X.
   릴레이션 내의 중복된 투플은 허용X.
   투플의 순서 상관X.         


2.1.3 기본키.
기본키 선정 시 고려사항.
고유한 값을 가진다. ( 중복X )
NULL 값은 허용X.
키값의 변동은 발생X.
문제발생 소지가 없어야 함.

릴레이션 스키마를 표현할 때 기본키는 밑줄을 그어서 표시함.
-> 릴레이션 이름 ( 속성1 , 속성2, .... , 속성N )
                        ------

2.1.6 외래키.
다른 릴레이션의 기본키를 참조 하는 속성.
관계형 DB -> Table 간의 관계.
양쪽 릴레이션의 도메인은 서로 같아야 함.
NULL , 중복값 허용.
기본키의 일부가 될수 있음.
